Report

Effective communication

What is the specified purpose of the site?

The purpose of the site is to provide information on the rochdale pioneer explorer scout unit

What information is the site attempting to communicate?

The site is attempting to provide information on all aspects of the explorer scout movement

Types of users identified for the site?

The site is aimed at people looking for information about scouting in general as well as new recruits in the rochdale area. The site is aimed at current and new members.

What interactive features could usefully be added to the site?

The site could have an interactive contact form added as well as a member portal for users to create a profile and post messages from

Site design, ease of navigation and accessibility

How I decided the structure of the site?

The site navigational structure was based on the existing site which enabled easy access to all areas for the users. The simple single level navigation also makes it easier to add new pages.

How I organized the information to be put on individual pages?

The information on each page is contained within the main block of the page design. The text is mixed in with images to present a clean professional page to the end user

Consideration I gave to navigation and usability

The sites usability and navigation was designed to be simple and cross platform. The site was designed to be easy to navigate for users of all age groups and skill level. As the navigation is single level there is nothing complicated to maintain for navigate.

How I developed and tested the site

I started building a PSD of what I wanted the site to look like, I then sliced the image and spent hours writing CSS and JS. As I would normally spend a good month prototyping and getting user feedback on designs and features this site is no where near complete and has been rushed. The site may be used on the production setup after 1/2 month of ripping it apart to find all the cross browser JS and CSS issues and also building it into a php application to enable the interactivity of users on such things as member portals.

Page design, layout, use of fonts, use of images, use of color

How I decided the page layout for the site

The page layout is simple and effective, it has been proven to be liked by users while maintaining a professional look.

How I positioned the elements of the pages

I used a combination of fixed and variable with divs as well as tables depending which suited the application. The main design is built with divs and tables just plain suck and any semi decent developer ditched table layouts years ago.

Fonts and sizes I used and why

I used Calibri sans serif at 1em size for the main text because it looks professional and is easy to read. I also used a custom font in the images which is the official scout font, this was done to keep in with the color scheme and style of the page.

Graphic file format

I used a mix of png and gif images on the site depending which format suited the application. The site is mostly gif images but there are a few png images that are slightly higher quality or have transparent effects on them.

Page color scheme

The entire sites design is based around the colors of the explorer scout logo and branding. Everything from the font to the colors is identical to the branding. This maintains a professional image for the entire site as well as toying into the scouting organization effectively.

Web editing tools

It says to do this part on Nvu but I have never used it. I feel it is wrong to teach people how to do things using GUI/WYSIWYG editors without even touching on the code. I admit that applications such as dreamweaver do sometimes help on sites with very complex layouts etc but really for people starting out notepad is all thats needed. I think everyone should make an effort to understand the code and how things work not just accept that they work. I refuse to waste my disk space installing a piece of software that I will never ever used and that the job of it can already be done by multiple other applications which are much much better. I mean if your gonna distribute a GUI editor at least make it a nice decent opensource IDE not just a cruddy WYSIWYG editor.
